# 上下文切换实现版本说明

本项目提供了两种上下文切换的实现方式：

## 🔧 版本选择

### 1. **纯汇编版本** (默认)
- 文件：`src/context_switch.s`
- 特点：性能最优，代码最紧凑
- 适用：生产环境，追求极致性能

### 2. **C + 内联汇编版本**
- 文件：`src/context_switch.c`
- 特点：可读性更好，更容易理解和维护
- 适用：学习目的，代码维护优先

## 🔄 如何切换版本

### 使用 C + 内联汇编版本：

1. **重命名汇编文件**（避免冲突）：
```bash
mv src/context_switch.s src/context_switch.s.bak
```

2. **编译项目**：
```bash
make clean
make all
```

### 恢复汇编版本：

1. **删除 C 版本文件**：
```bash
rm src/context_switch.c
```

2. **恢复汇编文件**：
```bash
mv src/context_switch.s.bak src/context_switch.s
```

3. **重新编译**：
```bash
make clean
make all
```

## 📊 性能对比

| 特性 | 汇编版本 | C + 内联汇编版本 |
|------|----------|------------------|
| 执行速度 | 最快 | 稍慢（编译器优化影响） |
| 代码大小 | 最小 | 稍大 |
| 可读性 | 较低 | 高 |
| 维护性 | 较难 | 容易 |
| 调试难度 | 高 | 中等 |
| 移植性 | 低 | 较高 |

## ⚠️ 注意事项

1. **功能等价**：两个版本实现完全相同的功能
2. **接口一致**：函数签名和调用方式完全相同
3. **不能同时存在**：同一时间只能使用一个版本
4. **编译器要求**：C 版本需要支持 GCC 内联汇编语法

## 🎯 推荐使用场景

- **学习 RTOS 原理**：使用 C + 内联汇编版本
- **生产环境部署**：使用纯汇编版本
- **代码审查和维护**：使用 C + 内联汇编版本
- **性能关键应用**：使用纯汇编版本
